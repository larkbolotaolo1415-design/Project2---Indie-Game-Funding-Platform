# LaroFund - Indie Game Funding Platform

A Stellar-based funding platform for Filipino indie games that issues tokenized game equity and distributes USDC revenue shares via Soroban.

## Problem

Indie Filipino game developers struggle to raise development funding and lack a transparent path for backers to invest in games. Traditional funding is opaque and centralized, while backers want a fair way to share in revenue. LaroFund solves this by turning game equity into on-chain tokens and automating revenue payout.

## How It Works

Users connect a Freighter wallet on Testnet, fund their account, and add trustlines for USDC and the game equity asset (`LARO`). Developers deploy the Soroban `LaroFund` contract, distribute USDC revenue into the contract, and backers claim their proportional share based on `LARO` holdings. The frontend reads contract state via Soroban RPC and builds signed transactions in Freighter.

## How It Uses Stellar

- Classic Assets for tokenized game equity (`LARO`) and demo USDC
- Trustlines to hold non-native assets
- Soroban smart contract for:
  - `init` to configure contract state
  - `distribute` to deposit USDC and update revenue-per-share
  - `claim` to withdraw earned revenue
  - `get_info` to power the UI dashboard
- Soroban RPC simulation and transaction assembly
- Freighter wallet integration for signing transactions
- Horizon and RPC for account/balance and contract interactions

## Setup & Run

```bash
git clone [your repo]
cd "Project2 - Indie Game Funding Platform"
cd web
npm install
npm run dev
```

From the repo root, deploy the Soroban contract:

```powershell
.\scripts\deploy.ps1
```

The deploy script writes:
- `NEXT_PUBLIC_CONTRACT_ID`
- `NEXT_PUBLIC_USDC_ID`
- `NEXT_PUBLIC_EQUITY_ID`

into `web/.env.local`.

## Network Details

- Network: testnet
- RPC URL: `https://soroban-testnet.stellar.org`
- Contract IDs: set in `web/.env.local` after deploy
- Asset issuers: generated by `scripts/deploy.ps1` and stored in `web/.env.local`
